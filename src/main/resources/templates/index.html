llm-index

A local knowledge graph index for your repo

Directory tree, class/method skeletons, dependencies, and a queryable graph — built so an LLM (or you) can find the right code without reading the whole repo.

01

Context-window friendly

Generates targeted markdown — skeletons and call chains, not the whole repo — so you paste less into a prompt.

02

Queryable dependency graph

Maps what a class uses and what uses it, so you can see the blast radius of a change before you make it.

03

Zero-config for Java

Works out of the box on Java 21+ / Maven projects. Run it locally via CLI or use the browser version below.

How it works

Four steps, no configuration: point it at code, it parses every file, writes the index, then you search it.

01

Point it at code

Run the CLI against a local project, or paste a git URL into the browser version above.

02

Parse every file

JavaParser walks each .java file and extracts classes, fields, methods, and Spring annotations.

03

Build the index

Writes tree/skeleton/dependency markdown plus a SQLite graph of USES, CALLS, CONTAINS, and EXPOSES edges.

04

Query it

Search a class or keyword and get matches, what depends on it, and its outward call chain — ready to paste into an LLM.

Example output

Querying OrderService after indexing returns something like this — small enough to paste directly into a prompt.

# Context for: OrderService

## Matching code
- OrderService (src/main/java/.../service/OrderService.java:14)

## Used by (impact if changed)
- OrderController
- OrderScheduler

## Calls outward from here
  PaymentClient
  InventoryService.reserve

## Instructions
Read only the files listed above. Start at the file:line refs.

What you get

Every build writes these into .llm-index/ next to your project.

  • 01-tree.mdDirectory structure, filtered to source files
  • 02-skeleton.mdEvery class's fields and method signatures, with line numbers
  • 03-dependencies.mdWhich classes use which — a flat dependency list
  • graph.dbSQLite knowledge graph (nodes/edges) you can query repeatedly
  • context.mdWritten on each query — matches, impact, and call chain for one term
  • INSTRUCTIONS.mdTells the LLM how to read the rest, so it doesn't ingest the whole repo

Get started

>_

Run it locally

Have Maven and Java 21+? Build the jar and point it at any project on your machine. The index is written straight into that project's .llm-index/ folder. 

./mvnw -q -DskipTests clean package
java -jar target/indexer-*.jar build .
java -jar target/indexer-*.jar query MyClassName

First command builds it, second indexes the current directory, third searches the graph and writes context.md.

⋮⋮

Or point us at a repo

No local setup needed. Paste a public git URL and we'll clone it, index it here, and give you a browsable, searchable result.

Only http(s) URLs are accepted. Repos are cloned to a temporary workspace and discarded after 30 minutes.

Java codebases only, for now. Source: com.llm.indexer.
